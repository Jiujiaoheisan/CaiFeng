# 裁档 · 衣样图录 — 打包成平板 App 操作指南

## 这份压缩包里有什么
```
pwa/
├── index.html          App 主页面
├── app.js               App 逻辑代码
├── manifest.json         告诉系统"这是个App"的配置文件（名称、图标、全屏模式等）
├── service-worker.js      让App能离线启动、满足"可安装"条件的脚本
└── icons/                各种尺寸的App图标（已经做好，不用管）
```

整体效果：这是一个**离线优先**的衣样档案管理工具，左边管理图片、右边填写归属人/布料/日期/结算金额等数据，所有数据都存在设备本地，不联网也能用。

---

## 第一步：发布到 GitHub Pages（免费、几分钟搞定）

PWABuilder 需要扫描一个**公开网址**才能帮你生成 apk，所以第一步是把这几个文件放到网上。

1. 打开 [github.com](https://github.com)，登录你的账号（没有就先免费注册一个）
2. 右上角点 `+` → `New repository`
3. 仓库名随便起，比如 `caidang-app`，设为 **Public（公开）**，点 `Create repository`
4. 进入新建好的仓库页面，点 `uploading an existing file`（或者 `Add file` → `Upload files`）
5. 把压缩包解压后，**把 `pwa` 文件夹里的所有文件和 `icons` 文件夹一起拖进去上传**（保持目录结构：`icons` 文件夹要原样上传，不要拆散）
6. 滚到底部点 `Commit changes`
7. 上传完成后，点仓库页面上方的 `Settings` → 左侧菜单找到 `Pages`
8. 在 `Branch` 那里选择 `main`，文件夹选 `/ (root)`，点 `Save`
9. 等 1-2 分钟，刷新这个 Pages 设置页面，顶部会出现一行绿色提示，给你一个网址，形如：
   ```
   https://你的用户名.github.io/caidang-app/
   ```
   这就是你的公开网址，**先点开确认页面能正常打开、能看到App界面**。

---

## 第二步：用 PWABuilder 生成 apk

1. 打开 [www.pwabuilder.com](https://www.pwabuilder.com)
2. 在首页的输入框里粘贴你刚才那个 GitHub Pages 网址，点 `Start`
3. PWABuilder 会自动扫描这个网址，检测到 manifest 和 Service Worker 都已经配置好（应该会显示绿色的"Installable"通过标志）
4. 点击 `Package for stores`（或类似"打包"按钮）
5. 选择 **Android** 平台
6. 保持默认选项即可（包名、版本号等都有合理默认值，也可以按需修改包名，比如改成 `com.yourname.caidang`）
7. 点击生成，等待片刻，会得到一个可以下载的 `.apk` 文件（也可能打包成 `.zip` 里面含 apk，解压一下就有）

---

## 第三步：安装到平板

1. 把生成的 `.apk` 文件传到平板上（微信传输助手、QQ、U盘、邮箱都行）
2. 平板的"设置"里找到「安全」或「应用」选项，**临时允许"安装未知来源应用"**（具体路径不同品牌平板略有差异，一般叫"未知来源" / "允许安装其他来源应用"）
3. 在平板的文件管理器里找到这个 apk 文件，点开，按提示安装
4. 安装完成后，桌面上会出现"裁档"的图标，点开就是全屏运行，没有浏览器地址栏

---

## 重要提醒

- **数据存储位置**：所有衣样图片和记录数据都存在这个 App 自己的本地存储空间里，不会因为关闭App或重启平板丢失。但如果**卸载这个App**，数据会被一起删除——记得用App里"导出备份"功能定期导出 JSON 备份文件，存到平板其他地方或云盘。
- **以后想更新功能**：如果之后还要改App的某个功能，流程是：改好 `index.html`/`app.js` → 重新上传到同一个 GitHub 仓库覆盖旧文件 → 重新用 PWABuilder 生成一次新的 apk → 重新安装（新apk可以直接覆盖安装旧版本，不会丢失里面的数据，只要包名没变）。
- **如果 PWABuilder 提示某些检查没通过**：截图发给我，我可以根据具体报错调整配置文件。
